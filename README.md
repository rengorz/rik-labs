## Лабораторна робота 1 — Windows Presentation Foundation

**Опис:** Розробка десктопного застосунку з графічним інтерфейсом на платформі .NET з використанням WPF (Windows Presentation Foundation).

**Варіант 1 — Форма опитування.** Застосунок представляє користувачу питання по одному, приймає текстові відповіді та зберігає результати у текстовий файл на диску.

**Як працює:** При запуску відображається перше питання з прогрес-баром. Користувач вводить відповідь у текстове поле та переходить до наступного питання кнопками «Назад» / «Далі». Пропустити питання без відповіді неможливо. Після останнього питання результати автоматично зберігаються у файл `survey_YYYY-MM-DD_HH-mm-ss.txt` на робочому столі.

```bash
cd lab1
dotnet run
```

## Лабораторна робота 2 — Electron Framework

**Опис:** Розробка кросплатформного десктопного застосунку з використанням фреймворку Electron (Node.js + Chromium). Інтерфейс реалізовано в окремому renderer-процесі, взаємодія з файловою системою — через main-процес за допомогою IPC.

**Варіант 1 — Форма опитування.** Застосунок реалізує покрокове опитування користувача із збереженням відповідей у файл.

**Як працює:** Renderer-процес відображає питання та керує навігацією. При завершенні опитування через `ipcRenderer.invoke` викликається main-процес, який відкриває нативний діалог збереження файлу (`dialog.showSaveDialog`) і записує результати. Архітектура дотримується принципу `contextIsolation` через `preload.js`.

```bash
cd lab2
npm install
npm start
```

## Лабораторна робота 3 — Qt Framework

**Опис:** Розробка застосунку з використанням фреймворку Qt. Мова програмування — Python, бібліотека PyQt6.

**Варіант 1 — Форма опитування.** Застосунок реалізує послідовне опитування користувача з нативним Qt-інтерфейсом та збереженням результатів у файл.

**Як працює:** Головне вікно (`QMainWindow`) містить прогрес-бар (`QProgressBar`), мітку з питанням та багаторядкове поле вводу (`QTextEdit`). Логіка збереження винесена в окремий модуль `file_saver.py`. Діалог збереження файлу реалізовано через `QFileDialog.getSaveFileName`.

```bash
cd lab3
pip install -r requirements.txt
python main.py
```

## Лабораторна робота 4 — Mobile Application (Android)

**Опис:** Розробка мобільного застосунку для платформи Android. Мова програмування — Kotlin, нативний Android SDK з використанням View Binding та ViewModel.

**Варіант 1 — Форма опитування.** Мобільний застосунок проводить опитування користувача та зберігає відповіді у текстовий файл на пристрої.

**Як працює:** Стан опитування зберігається у `SurveyViewModel` (переживає rotate екрану). `MainActivity` рендерить поточний стан через View Binding. Файл зберігається в app-specific external storage (`getExternalFilesDir`) без необхідності запитувати дозволи на Android 10+. Відкривається у Android Studio та запускається на емуляторі або фізичному пристрої.

```
Відкрити папку lab4/ в Android Studio → Run 'app'
```

## Лабораторна робота 5 — React Application

**Опис:** Розробка веб-застосунку з використанням бібліотеки React. Збірка виконується через Vite. Тема застосунку — **Task Tracker** (менеджер завдань).

**Як працює:** Застосунок складається з кореневого компонента `App`, кастомного хука `useTasks` (управління станом + `localStorage`) та чотирьох компонентів: `TaskInput`, `TaskFilter`, `TaskList`, `TaskItem`. Підтримує фільтрацію завдань за статусом (Усі / Активні / Виконані), видалення окремих завдань та очищення виконаних. Дані зберігаються між сесіями через `localStorage`.

```bash
cd lab5
npm install
npm run dev
```

## Лабораторна робота 6 — Single Page Application

**Опис:** Розробка веб-застосунку з реалізацією парадигми SPA (Single Page Application). Реалізовано на чистому JavaScript (без фреймворків) з власним hash-роутером. Тема застосунку — **Finance Tracker** (облік фінансів).

**Як працює:** Навігація між сторінками здійснюється через зміну hash-частини URL (`#/`, `#/transactions`, `#/add`) без перезавантаження сторінки. Роутер (`router.js`) слухає подію `hashchange` і викликає відповідний обробник. Стан зберігається у `localStorage` через модуль `store.js`. Три сторінки: **Огляд** (зведення балансу, доходів, витрат + останні транзакції), **Транзакції** (повний список з фільтром і видаленням), **Додати** (форма з вибором типу дохід/витрата, категорії та суми).

```bash
cd lab6
npx serve .
```
